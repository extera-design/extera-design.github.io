# Extera's Project Manager REST API

Extera’s Project Manager API (Application Programming Interface) power its platform for CFD (Computational Fluid Dynamics). Behind this API is a software layer providing access to Extera projects and allowing to run and capture results from individual cases.

# So, what’s a REST API?

An API (Application Programming Interface) is a set of rules that lets programs talk to each other, exposing data and functionality across the internet in a consistent format.

REST (Representational State Transfer) is the architectural pattern that describes how distributed systems can expose a consistent interface. When people use the term ‘REST API,’ they are generally referring to an API accessed via HTTP protocol at a predefined set of URLs.

These URLs represent various resources - any information or content accessed at that location, which can be returned as JSON, HTML, audio files, or images. Often, resources have one or more methods that can be performed on them over HTTP, like GET, POST, PUT and DELETE. Behind the scenes the REST API may also perform complex tasks such as running a case, or postprocessing.

# Authentication

All API (other than the Authentication API) use the Bearer (opens new window) authentication schema with a JWT token (opens new window)

Example

Authorization: Bearer eyJhbGciOiJI ... vihbfd1A

Before calling on any of the APIs, the user must retrieve a token from the Authentication API. Unauthenticated calls will be rejected.

# Websocket

The REST API is accompanied by a websocket. For users of the API, the protocol for using the websocket is as follows:

An example of the subscribe message payload is:

{
  "subscribe": true,
  "auth": "eyJhbGciOiJ ... wWcKwT8Ep3Wo4"
}

Each API call provides both a response and a notification on the websocket to all authenticated listeners.

An example notification is:

{
  "type": "Case",
  "action": "changed",
  "result": {
    "_id": "60a3236e383bb9790cd46852",
    "name": "Case #1",
    "owner": "60907b23eddd2e1ef4270f9b",
    "parent": "60a2d2791f6f74740c00b7a5"
  }
}
